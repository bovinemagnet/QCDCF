= API Model Reference
Paul Snow
:description: Detailed reference for all types in com.paulsnow.qcdcf.model and the EventSink interface, including ChangeEnvelope, OperationType, CaptureMode, TableId, RowKey, SourcePosition, WatermarkContext, and PublishResult.

This page documents every type in the `{base-package}.model` package and the `EventSink` interface from `{base-package}.core.sink`.
These types form the stable boundary contract of the entire {project-name} pipeline.
All modules — source adapter, core engine, sink implementations — exchange data exclusively through these types.

NOTE: The `cdc-api-model` module has no Quarkus, CDI, or persistence dependencies.
Changes to any type on this page are breaking changes for all modules.

== ChangeEnvelope

`com.paulsnow.qcdcf.model.ChangeEnvelope`

The canonical change event record.
Every change detected by the pipeline — whether from the PostgreSQL write-ahead log, a snapshot chunk read, or the reconciliation engine — is wrapped in a `ChangeEnvelope` before delivery to a sink.

[source,java]
----
public record ChangeEnvelope(
        UUID eventId,
        String connectorId,
        TableId tableId,
        OperationType operation,
        CaptureMode captureMode,
        RowKey key,
        Map<String, Object> before,
        Map<String, Object> after,
        SourcePosition position,
        Instant captureTimestamp,
        WatermarkContext watermark,
        Map<String, String> metadata
)
----

=== Fields

`eventId`::
A `UUID` that uniquely identifies this event.
Generated by the framework on creation.
Stable across at-least-once replay — the same logical change may produce duplicate events with different `eventId` values after a WAL replay.
Use `position.lsn` and `key` together for idempotent deduplication, not `eventId` alone.

`connectorId`::
The identifier of the connector instance that produced this event.
Matches the value of the `qcdcf.connector.id` configuration property.
Useful for routing and filtering in multi-connector deployments.

`tableId`::
A `TableId` record identifying the source table.
See <<tableid>>.

`operation`::
The type of change.
An `OperationType` enum value.
See <<operationtype>>.

`captureMode`::
How the event was captured.
A `CaptureMode` enum value.
See <<capturemode>>.

`key`::
The primary key of the affected row, as a `RowKey` record.
Never null.
See <<rowkey>>.

`before`::
Column values before the change, as an unmodifiable `Map<String, Object>`.
`null` for `INSERT` and `SNAPSHOT_READ` operations.
Map values may themselves be `null` for database `NULL` column values.

`after`::
Column values after the change, as an unmodifiable `Map<String, Object>`.
`null` for `DELETE` operations.
Map values may themselves be `null` for database `NULL` column values.

`position`::
The `SourcePosition` identifying this event's place in the PostgreSQL WAL.
`null` for snapshot-only events that have no WAL counterpart.
See <<sourceposition>>.

`captureTimestamp`::
The `Instant` at which the framework captured this event.
Not the same as the database commit timestamp, which is in `position.commitTimestamp`.

`watermark`::
The `WatermarkContext` associating this event with a snapshot window.
`null` for events that are not part of a snapshot reconciliation window.
See <<watermarkcontext>>.

`metadata`::
An unmodifiable `Map<String, String>` for extensible tracing headers, schema version identifiers, or any other string-valued properties.
Never `null`; defaults to an empty map.

=== Constructor Constraints

The compact constructor enforces the following:
`eventId`, `connectorId`, `tableId`, `operation`, `captureMode`, `key`, and `captureTimestamp` must all be non-null.
`before`, `after`, and `metadata` may be null on construction; `before` and `after` are stored as null, whilst `metadata` is stored as an empty map.

[[operationtype]]
== OperationType

`com.paulsnow.qcdcf.model.OperationType`

Describes the type of change represented by a `ChangeEnvelope`.

[source,java]
----
public enum OperationType {
    INSERT,
    UPDATE,
    DELETE,
    SNAPSHOT_READ,
    HEARTBEAT
}
----

`INSERT`::
A new row was inserted into the source table.
`before` is `null`; `after` contains the new row values.
Produced with `CaptureMode.LOG`.

`UPDATE`::
An existing row was modified.
`before` contains the previous values if the table's replica identity is set to `FULL`; otherwise `before` contains only the primary key columns.
`after` contains the new values.
Produced with `CaptureMode.LOG`.

`DELETE`::
An existing row was deleted.
`before` contains the row values if replica identity is `FULL`, otherwise only primary key columns.
`after` is `null`.
Produced with `CaptureMode.LOG`.

`SNAPSHOT_READ`::
A row was read during a snapshot chunk operation.
`before` is always `null`.
`after` contains the row values as they were at the time the chunk query executed.
Produced with `CaptureMode.SNAPSHOT`.
If the row was also modified during the enclosing watermark window, the reconciliation engine removes this event in favour of the log event.

`HEARTBEAT`::
A periodic liveness event carrying no row data.
`key`, `before`, and `after` fields are not meaningful for heartbeat events.
Used for monitoring WAL consumer lag and advancing the replication slot position when no DML activity has occurred on tracked tables.

[[capturemode]]
== CaptureMode

`com.paulsnow.qcdcf.model.CaptureMode`

Indicates how a `ChangeEnvelope` was produced.
Complements `OperationType` by recording the capture pathway rather than the change semantics.

[source,java]
----
public enum CaptureMode {
    LOG,
    SNAPSHOT,
    RECONCILED
}
----

`LOG`::
The event was captured directly from the PostgreSQL write-ahead log via the logical replication stream.
The `position` field will be populated.
Used for `INSERT`, `UPDATE`, `DELETE`, and `HEARTBEAT` operations.

`SNAPSHOT`::
The event was captured during a chunked snapshot read.
The `position` field is `null` because chunk reads do not have a corresponding WAL position.
The `watermark` field will be populated with the enclosing window context.
Used for `SNAPSHOT_READ` operations.

`RECONCILED`::
The event was produced by the reconciliation engine as a result of merging log and snapshot data.
A `RECONCILED` event carries the merged, authoritative state after the collision-removal algorithm has run.
Reserved for future use in the current release; the reconciliation engine currently emits `LOG` and `SNAPSHOT` events directly.

[[tableid]]
== TableId

`com.paulsnow.qcdcf.model.TableId`

Identifies a database table by schema and name.

[source,java]
----
public record TableId(String schema, String table)
----

`schema`::
The PostgreSQL schema name, for example `public` or `accounts`.
Never `null`.

`table`::
The table name, for example `customer` or `order_line`.
Never `null`.

`canonicalName()`::
Returns the fully qualified name as `schema.table`, for example `public.customer`.
This form is used in log output, metrics labels, Kafka topic name construction, and anywhere a single unambiguous string identifier for a table is needed.

[source,java]
----
TableId id = new TableId("public", "customer");
assert id.canonicalName().equals("public.customer");
----

`toString()` delegates to `canonicalName()`.

[[rowkey]]
== RowKey

`com.paulsnow.qcdcf.model.RowKey`

Represents the primary key of a captured row as an ordered map of column names to values.

[source,java]
----
public record RowKey(Map<String, Object> columns)
----

`columns`::
An unmodifiable, alphabetically sorted `Map<String, Object>` of primary key column names to their values.
Never `null` and never empty.
Values are the Java types produced by the JDBC driver for the corresponding PostgreSQL column type.

The constructor sorts column entries alphabetically by column name and stores an unmodifiable copy.
This ensures that `RowKey` instances constructed from the same column set in any order are equal under `equals()` and produce the same `toCanonical()` string.

`toCanonical()`::
Returns a deterministic string representation suitable for hashing, comparison, and use as a map key.
Columns are sorted alphabetically; values use their `toString()` representation.
The format is `col1=val1,col2=val2`.

[source,java]
----
// Single-column primary key
RowKey single = new RowKey(Map.of("id", 42));
assert single.toCanonical().equals("id=42");

// Composite primary key — order of construction does not matter
RowKey composite = new RowKey(Map.of("tenant_id", "acme", "order_id", 99));
assert composite.toCanonical().equals("order_id=99,tenant_id=acme");
----

The reconciliation engine uses `toCanonical()` to build the set of log-event keys for collision detection against snapshot chunk rows.

[[sourceposition]]
== SourcePosition

`com.paulsnow.qcdcf.model.SourcePosition`

Records the position of an event in the PostgreSQL write-ahead log.

[source,java]
----
public record SourcePosition(long lsn, Long txId, Instant commitTimestamp)
        implements Comparable<SourcePosition>
----

`lsn`::
The Log Sequence Number (LSN) of the WAL record.
An LSN is a 64-bit integer that monotonically increases within a PostgreSQL instance.
LSN ordering is the authoritative ordering for events from the same source instance.
Comparing LSNs from different instances is not meaningful.

`txId`::
The PostgreSQL transaction ID.
May be `null` for non-transactional events such as heartbeats.
Multiple events with the same `txId` belong to the same committed transaction.

`commitTimestamp`::
The wall-clock timestamp at which the transaction was committed.
Never `null`.
Note that commit timestamps from different transactions are not guaranteed to be unique; use `lsn` as the primary ordering key.

`compareTo(SourcePosition)`::
Implements natural ordering by `lsn` only.
This makes `SourcePosition` directly sortable into WAL order.

A convenience constructor `SourcePosition(long lsn, Instant commitTimestamp)` is provided for cases where the transaction ID is not available.

[[watermarkcontext]]
== WatermarkContext

`com.paulsnow.qcdcf.model.WatermarkContext`

Associates a change event with a snapshot reconciliation window.

[source,java]
----
public record WatermarkContext(UUID windowId, Instant lowMark, Instant highMark)
----

`windowId`::
A `UUID` that uniquely identifies this watermark window.
Assigned when the `SnapshotCoordinator` initiates a chunk read.
All events belonging to the same window share this ID.

`lowMark`::
The timestamp at which the low watermark was written to the source database.
Never `null`.
The low watermark write is an `UPDATE` to `cdc_control.watermark` that flows through the replication slot as a recognisable WAL event, signalling the start of the window.

`highMark`::
The timestamp at which the high watermark was written.
`null` while the window is still open — that is, while the snapshot chunk query is in progress.
Set when the high watermark `UPDATE` has been written and the WAL stream has caught up to that position.

`isClosed()`::
Returns `true` if `highMark` is non-null, indicating that the window has been closed and is ready for reconciliation.

=== Window Lifecycle

. The `SnapshotCoordinator` writes the low watermark and creates a `WatermarkContext` with `highMark = null`.
. The chunk query executes.
Rows read during this step carry a `WatermarkContext` with `highMark = null`.
. The high watermark is written.
The `WatermarkContext` is updated with the `highMark` timestamp and `isClosed()` returns `true`.
. The `WatermarkAwareEventRouter` hands the closed window to the `ReconciliationEngine`.

See xref:ROOT:architecture.adoc#watermark-reconciliation-algorithm[Watermark Reconciliation Algorithm] for the full algorithmic description.

== EventSink

`com.paulsnow.qcdcf.core.sink.EventSink`

The primary output interface of the CDC pipeline.
All `ChangeEnvelope` records produced by the reconciliation engine are delivered through an `EventSink` implementation.
The `cdc-core` module depends only on this interface; all broker-specific code resides in sink adapter modules.

[source,java]
----
public interface EventSink {

    PublishResult publish(ChangeEnvelope event);

    PublishResult publishBatch(List<ChangeEnvelope> events);
}
----

`publish(ChangeEnvelope event)`::
Publishes a single change event.
Returns a `PublishResult` indicating success or failure.
Implementations must not throw; failures must be represented as `PublishResult.Failure`.

`publishBatch(List<ChangeEnvelope> events)`::
Publishes a batch of change events.
Implementations may process the events individually or deliver them as a single atomic batch to the underlying broker.
Returns a single `PublishResult` for the batch as a whole.
On failure, the delivery coordinator will replay the entire batch.

=== PublishResult

`com.paulsnow.qcdcf.core.sink.PublishResult`

A sealed interface representing the outcome of a publish call.

[source,java]
----
public sealed interface PublishResult {

    record Success(int eventCount) implements PublishResult {}

    record Failure(String reason, Throwable cause) implements PublishResult {}

    default boolean isSuccess() {
        return this instanceof Success;
    }
}
----

`Success(int eventCount)`::
Indicates that the publication succeeded.
`eventCount` is the number of events that were durably acknowledged by the sink.

`Failure(String reason, Throwable cause)`::
Indicates that the publication failed.
`reason` is a human-readable explanation.
`cause` is the originating exception; may be `null` if no exception was thrown.
A convenience constructor `Failure(String reason)` is available when there is no underlying exception.

`isSuccess()`::
Default method returning `true` if the result is a `Success` instance.

=== MVP Implementations

`LoggingEventSink`::
Logs each event at `DEBUG` level using structured log fields.
Does not deliver events to any external system.
Returns `PublishResult.Success` for every call.
Suitable for development and smoke-testing the pipeline.

`InMemoryEventSink`::
Appends events to an in-memory `List<ChangeEnvelope>`.
Returns `PublishResult.Success` for every call.
Suitable for integration tests that need to assert on the exact sequence of events emitted by the pipeline.
